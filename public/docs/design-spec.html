01

Product Overview

🎨
Art-first philosophy
Unlike performance-focused ad tools, RiveAds prioritizes aesthetic quality and creative expression. Brands get ads that feel designed, not generated.
🤖
AI as creative partner
AI handles the heavy lifting — layout, animation logic, copy — while always leaving the door open for manual control at every level.
⚡
Rive-native output
Built around Rive's runtime for buttery-smooth, interactive, hardware-accelerated animations. Outputs embed cleanly in any web environment.
📦
Multi-format export
One creation → three deployable formats: HTML/JS ad unit, MP4/GIF video, and a static image fallback for legacy placements.
🔧
Progressive control
Users can start with a one-line prompt, refine via AI chat, and drop into manual fine-tuning — as deep or shallow as they want.
🚀
Future-ready scope
Architecture designed to extend to video ads, social posts, OOH, and other brand materials beyond display advertising.
02

Target Users

Primary User
Brand Creative Teams
In-house designers, creative directors, and brand managers at companies who control their own visual identity. They have taste and a brand system — they need speed and interactivity without outsourcing to an agency.
Has brand guidelines Wants creative control Time-constrained
Secondary User
Independent Advertisers
Founders, marketing managers, and indie brands who lack a creative team but want ads that feel premium. They rely more heavily on the AI and guided wizard flows, and value smart defaults.
Less design experience Needs strong defaults Budget-conscious
Non-user (explicit): Performance marketers optimizing purely for CTR, ROAS, and conversion. This product is intentionally not for them — the positioning is artistic quality over conversion mechanics.

Core User Flow

Click each step to expand details.

1
Brand Setup (one-time)
Onboarding · ~2 min
User uploads brand assets: logo, color palette, fonts, optional style references (moodboard images). The AI ingests these to establish a brand token system used across all future creations. This step is skippable for users who want to start immediately.
Brand tokensMoodboard uploadSkippable
2
Choose Creation Mode
Mode Selection · ~10 sec
User picks between two paths: (A) Chat Prompt — type a freeform brief like "a banner for our summer fragrance launch, dreamy and slow-motion"; or (B) Guided Wizard — step-by-step questions on format, mood, copy, and animation style. Both modes converge to the same editor.
Chat modeWizard mode
3
AI Generation
Automated · ~10–30 sec
AI generates 2–3 creative variations using the Rive runtime: layout, animation sequences, copy placement, color application from brand tokens. Each variation is shown as a live preview with its Rive animation playing. Users pick a starting point or remix a variation.
2–3 variantsLive previewRive-powered
4
Refinement (AI or Manual)
Editor · Variable time
The main editor workspace. Users can refine via (a) AI chat panel — "make the animation slower and more ethereal", "swap the background to a deep navy" — or (b) direct manual controls: layer panel, property inspector, timeline scrubber for animation keyframes. Both modes are available simultaneously.
AI chat panelManual controlsTimeline
5
Export & Deploy
Output · ~1 min
User selects one or more export formats: HTML/JS embeddable ad unit (with standard IAB sizes), MP4/GIF video render, or static PNG/JPG fallback. Each format is generated automatically. User gets a copy-paste embed code, direct download, or shareable preview link.

Key Screens

Describe your ad...
Creation Entry — Chat Mode
Clean, distraction-free prompt interface. A large input field with example prompts. Users describe the ad in their own words; the AI asks clarifying questions if needed before generating.

Technical Architecture

Frontend
React + TypeScript
Component-based editor UI; strong ecosystem for complex state management
Animation Runtime
Rive Web Runtime
Official @rive-app/canvas package for live preview and HTML/JS output embedding
AI / LLM
Claude API (Anthropic)
Creative direction, prompt interpretation, property generation, refinement chat
Rive Generation
Rive API / Editor SDK
Programmatic creation and manipulation of .riv files from AI-generated specs
Video Render
Puppeteer + FFmpeg
Server-side headless render of Rive canvas to MP4/GIF via frame capture
Backend
Node.js + tRPC
Type-safe API layer; handles generation jobs, export queue, asset storage
Storage
S3 + PostgreSQL
.riv files and exports in S3; project metadata, brand tokens in Postgres
Auth
Clerk or Auth.js
Fast integration, supports social login and org/team accounts for future collab
Key Technical Risk
Programmatic Rive file generation is the core technical challenge — the Rive Editor SDK's API surface for headless creation is limited. Early prototyping should validate whether .riv files can be generated from AI-described specs, or whether the approach should use a library of template .riv files that the AI parameterizes.

Open Questions

Q1
How will the AI generate .riv files — from templates or from scratch?
The most important architectural decision. Template-based (parameterize pre-built .riv files) is faster to build; scratch generation is more creative but technically harder. Consider a hybrid: AI selects and adapts from a curated template library.
Q2
What is the pricing model?
Per-generation credits, monthly subscription by seat, or flat-rate unlimited? Video rendering is expensive server-side — needs to be factored into unit economics.
Q3
What IAB ad sizes will be supported at launch?
Recommend starting with the top 5: Leaderboard (728×90), Medium Rectangle (300×250), Large Rectangle (336×280), Half Page (300×600), Billboard (970×250).
Q4
Who owns the Rive file IP — the user or the platform?
Important for ToS and for any potential marketplace of templates in the future. If users can export .riv source files, they can import them into Rive's own editor — is that desirable?
Q5
Will there be a template marketplace or style library?
A library of curated animation styles (e.g., "liquid", "geometric", "typographic") would accelerate quality and reduce generation complexity. Could also be a monetization lever.
Q6
How is the "artistic" positioning enforced or measured?
Consider an internal quality bar — e.g., AI-generated outputs are reviewed against aesthetic benchmarks before being surfaced to users. This protects the brand positioning.
